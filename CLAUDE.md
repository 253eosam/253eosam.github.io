# Claude 기반 공부 노트 워크플로우

이 프로젝트는 Claude를 활용하여 공부한 내용을 체계적으로 정리하는 개인 학습 노트 공간입니다. 공부한 내용을 상세하게 기록하고, 나중에 다시 볼 때 빠르게 이해할 수 있도록 구조화된 형태로 관리합니다.

## 목적

- 공부한 내용을 빠짐없이 상세하게 정리
- 다시 볼 때 빠르게 이해할 수 있는 구조화된 노트 작성
- Claude와의 대화를 통한 개념 정리 및 이해도 심화
- 학습 내용의 체계적 관리 및 아카이브
- 위키/지식베이스 형태의 읽기 쉬운 레퍼런스 구축

## 빠른 시작

### 자동화 스크립트

모든 스크립트는 프로젝트 루트에서 실행합니다.

```bash
# 새 노트 시작 (content/in-progress/에 생성)
./tools/new.sh "토픽명"

# 노트 완성 (in-progress → ready-to-publish, 메타데이터 자동 추출)
./tools/done.sh "폴더명" [품질점수]

# 발행 처리 (ready-to-publish → published)
./tools/publish.sh "폴더명" "https://velog.io/@253eosam/..."

# 실시간 현황 확인
./tools/status.sh

# 전체 노트 레지스트리 생성
./tools/generate-registry.sh
```

## 워크플로우 및 폴더 구조

### 노트 작성 플로우

1. **새 노트 시작** (`./tools/new.sh "토픽명"`)
   - `content/in-progress/YYYY-MM-DD-<토픽명>/`에 작업 폴더 생성
   - `content/templates/draft-template.md` 기반 draft.md 자동 복사
   - Draft 작성 + Claude AI와 대화하며 내용 정리 및 보완

2. **노트 완성** (`./tools/done.sh "폴더명" [품질점수]`)
   - `content/in-progress/` → `content/ready-to-publish/`로 이동
   - frontmatter 파싱 → metadata.json 자동 생성
   - 최종 검토 및 메타데이터 완성

3. **발행** (`./tools/publish.sh "폴더명" [velog-url]`)
   - 블로그에 발행
   - `content/ready-to-publish/` → `content/published/`로 이동

### 폴더 구조

```
/
├── content/
│   ├── templates/            # Draft 템플릿 보관
│   │   └── draft-template.md
│   ├── in-progress/          # 작성 중인 노트 (new.sh로 생성)
│   ├── ready-to-publish/     # 정리 완료된 노트 (done.sh)
│   ├── published/            # 발행 완료된 노트 (publish.sh)
│   └── posts/                # 기존 노트 (개선 대상)
├── content-registry.json     # 전체 노트 현황 (자동 생성)
└── tools/                   # 자동화 스크립트
```

### 워크플로우

#### 기존 노트 개선
```
content/posts/ → [rewrite] → content/in-progress/ → [done] → content/ready-to-publish/ → [publish] → content/published/
```

#### 신규 노트 작성
```
1. [new] → content/in-progress/에 작업 폴더 생성 (draft.md 포함)
2. content/in-progress/에서 공부 + Claude AI와 대화하며 내용 정리
3. [done] → content/in-progress/ → content/ready-to-publish/
4. [publish] → content/ready-to-publish/ → content/published/
```

## 노트 작성 가이드라인

### 노트 구조 원칙

모든 공부 노트는 다음 구조를 기본으로 한다. 다시 볼 때 빠르게 핵심을 파악할 수 있도록 위키/지식베이스 스타일로 작성한다.

#### 필수 구성 요소

1. **목차 (TOC)**
   - 글 상단에 목차를 배치하여 전체 구조를 한눈에 파악
   - 섹션 링크를 통해 원하는 부분으로 바로 이동 가능하게 구성

2. **개요 / 한줄 요약**
   - 이 주제가 무엇인지 1-2문장으로 정의
   - "이 개념을 왜 알아야 하는가"를 간결하게 명시

3. **본문 (상세 내용)**
   - 공부한 모든 내용을 빠짐없이 상세하게 기록
   - 개념 → 원리 → 예시 → 활용 순서로 논리적 흐름 유지
   - 코드 예시는 실제 동작하는 완전한 코드로 작성
   - 비교 대상이 있으면 표로 정리하여 차이점을 명확히

4. **다이어그램 / 시각 자료**
   - 복잡한 흐름이나 구조는 Mermaid 다이어그램 또는 ASCII art로 시각화
   - 텍스트만으로 이해하기 어려운 부분은 반드시 그림으로 보완
   - 예시:
     ```mermaid
     graph LR
       A[입력] --> B[처리] --> C[출력]
     ```

5. **Q&A (질문 섹션)**
   - 공부하면서 떠오른 질문과 그에 대한 답변을 정리
   - 질문은 본문의 문맥상 어울리는 위치에 인라인으로 배치
   - 형식: `> **Q: 질문 내용**` 블록인용 사용
   - 해당 개념을 설명하는 섹션 바로 아래에 관련 Q&A 배치
   - 예시:
     ```markdown
     ## 클로저란?
     클로저는 함수와 그 함수가 선언된 렉시컬 환경의 조합이다...

     > **Q: 클로저는 메모리 누수를 일으키나?**
     > 클로저가 참조하는 변수가 GC되지 않으므로 주의가 필요하다.
     > 불필요한 참조는 null로 해제하는 것이 좋다.
     ```

6. **요약 (Summary)**
   - 글 하단에 핵심 내용을 bullet point로 간결하게 요약
   - 다시 볼 때 요약만 읽어도 전체 내용이 떠올릴 수 있도록 작성
   - 키워드 중심으로 정리

### 작성 스타일

#### 1. 위키/지식베이스 스타일

- **정의 우선**: 각 개념은 명확한 정의로 시작
- **계층적 구조**: 큰 개념 → 세부 개념 → 예시 순서로 깊어지는 구조
- **크로스 레퍼런스**: 관련 개념이 다른 노트에 있으면 참조 링크 추가
- **독립적 섹션**: 각 섹션을 독립적으로 읽어도 이해할 수 있게 작성

#### 2. 상세함의 기준

- **생략 금지**: "자세한 내용은 공식 문서 참고" 같은 생략 표현 사용하지 않음
- **모든 단계 기록**: 설정, 설치, 실행 과정을 한 단계도 빠짐없이 기록
- **용어 설명**: 처음 등장하는 전문 용어는 반드시 간단한 설명 추가
- **코드 전체**: 코드 스니펫이 아닌, 복사해서 바로 실행 가능한 완전한 코드 작성

#### 3. 가독성 원칙

- **짧은 문단**: 한 문단은 3-4문장 이내로 유지
- **표 활용**: 비교, 옵션, 설정값 등은 표로 정리
- **코드 블록**: 모든 코드에 언어 태그 명시 (```js, ```python 등)
- **굵은 글씨**: 핵심 키워드나 중요 개념은 **굵은 글씨**로 강조
- **들여쓰기**: 하위 개념은 들여쓰기로 계층 표현

#### 4. 톤 가이드라인

- **사실 중심**: 감정적 표현이나 주관적 평가 최소화
- **간결한 문체**: "~이다", "~한다" 체로 통일
- **과도한 이모지 사용 금지**: 적절한 수준으로 제한
- **마무리 멘트 금지**: "도움이 되었으면", "공유해주세요" 등 불필요한 마무리 멘트 추가하지 않음
- **불필요한 소감 섹션 금지**: "개인적인 생각", "마지막 한 마디", "앞으로 어떻게 될까" 등 추상적인 소감/전망 섹션 추가하지 않음

#### 5. 구조 품질

- **깨진 표 수정**: 마크다운 표 형식 정확히 작성
- **명확한 섹션**: 내용에 맞는 적절한 제목 구조 (h2 → h3 → h4 순서 준수)
- **읽기 쉬운 형식**: 표, 코드 블록, 인용문, 다이어그램 적절히 활용
- **논리적 흐름**: 읽는 사람 입장에서 자연스러운 정보 순서

## 품질 관리 체크리스트

### 노트 품질 기준
- 공부한 내용이 빠짐없이 기록되었는가
- 다시 볼 때 목차와 요약만으로 전체 내용을 떠올릴 수 있는가
- 복잡한 개념에 다이어그램이나 시각 자료가 포함되어 있는가
- 떠오른 질문이 문맥에 맞는 위치에 Q&A로 정리되어 있는가
- 코드 예시가 완전하고 바로 실행 가능한가

### 완성 전 체크리스트
- 오타 및 문법 검토 완료
- 메타데이터 완성도 확인
- 목차와 실제 섹션 구조가 일치하는지 확인
- 모든 전문 용어에 설명이 포함되어 있는지 확인
